清除git仓库的所有提交记录，成为一个新的仓库

记录一下

操作步骤

  1. 切换到新的分支
  1. 将所有文件添加到新仓库
  1. 提交跟踪过的文件
  1. 删除master分支
  1. 重命名当前分支为master
  1. 提交到远程master分支

git commit -m 与 git commit -am 的区别

【am】 就是 【add modify】 两个命令的合并

如果是新增的文件就必须使用第一步和第二步的命令； git commit -am ‘xxx’命令只能提交已经跟踪过且修改了的文件。
